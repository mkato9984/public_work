🚀 GitHub Codespaces で RAGシステムを起動する

📍 ステップ 1: GitHubリポジトリにアクセス

URL: https://github.com/mkato9984/public_work

上記リンクをクリックしてGitHubリポジトリを開いてください。

🖱️ ステップ 2: Codespacesを起動

GitHubページで以下の順序でクリック：

  1. [Code] ボタン（緑色）
  2. [Codespaces] タブ
  3. [Create codespace on main] ボタン

📂 ステップ 3: プロジェクトフォルダに移動

Codespacesが起動したら、ターミナルで以下を実行：

cd rag_project/rag_system_project/02_web_rag

🔑 ステップ 4: Google API Keyの設定

重要: 環境変数を設定してください：

export GOOGLE_API_KEY="your-actual-api-key"

⚠️ API Keyがないと画面が真っ白になる可能性があります

🌐 ステップ 5: アプリケーション起動

python web_app.py

または

python web_app_github.py

✅ ステップ 6: ブラウザでアクセス

Codespacesが自動的にポート5000を転送します：

  • 自動転送URL: https://[codespace-name]-5000.githubpreview.dev
  • ローカルアクセス: http://localhost:5000

🔧 画面が真っ白な場合のトラブルシューティング

考えられる原因:

  • ❌ GOOGLE_API_KEYが未設定
  • ❌ PostgreSQLサービスが起動していない
  • ❌ ポート転送の問題
  • ❌ JavaScriptエラー

解決手順:

  1. ログ確認: ターミナルでエラーメッセージを確認
  2. ヘルスチェック: curl http://localhost:5000/health
  3. ブラウザ開発者ツール: F12でコンソールエラー確認
  4. 別ブラウザでテスト: Chrome、Edge、Firefoxで確認

🛠️ 手動デバッグコマンド

問題が発生した場合、以下を順番に実行：

# PostgreSQL状態確認
sudo service postgresql status

# PostgreSQL再起動
sudo service postgresql restart

# 環境変数確認
echo $GOOGLE_API_KEY

# Python依存関係確認
pip list | grep -E "(flask|psycopg2|google)"

# ポート確認
netstat -tulpn | grep :5000

📋 正常動作の確認方法

アプリが正常に動作している場合、以下が表示されます：

  • ✅ RAGシステムのメイン画面
  • ✅ 文書アップロード機能
  • ✅ 質問応答機能
  • ✅ 文書一覧表示
